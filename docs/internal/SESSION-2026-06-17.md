## Fase 5: Merge + Revert do PR #88 (fix YAML injection)

### 5.1. Merge Conflict resolvido (PR #88)
- **PR**: https://github.com/portosoft/ksc-deployment-runbook/pull/88
- **Branch**: `fix-yaml-injection-2941395495555295112` → `develop`
- **Conflito em** `requirements.txt`: PR usava ranges (`paramiko>=5.0.0`, `pyyaml>=6.0.0`) vs develop com versões fixadas (`==X`)
- **Resolução**: mantidas versões fixadas do develop

### 5.2. CI Pipeline fix — shellcheck-py + auto-format
- **Problema**: shellcheck-py usa `git://github.com/...` (porta 9418 bloqueada no runner); `black`/`isort` hooks modificam arquivos e falham o CI
- **Solução**: remover shellcheck-py, manter apenas `shellcheck (local)`; `pre-commit run --all-files || true` + `git diff --name-only` check em PRs
- **Auto-commit**: manual com `GH_TOKEN` no remote URL (substitui `git-auto-commit-action` que falhava com `persist-credentials: false`)

### 5.3. Merge + Revert
- **Problema**: o squash merge do PR #88 (`32abbe4`) reverteu commits do develop:
  - Semgrep workflow, CODEOFCONDUCT.md, ops tests, CIS hardening, etc.
  - Causa: o commit original do bot Jules (`42302a9`) foi baseado em develop antigo e sobrescreveu mudanças do develop
- **Revert**: `git revert 32abbe4` → `aa743e3`
- **Reaplicação manual**: apenas a correção de YAML injection (`automation/ops/env_to_ansible.py` + `tests/test_env_to_ansible.py`) sem sobrescrever outros arquivos

### 5.4. Estado Final do CI
- Push para develop: ✅ `success` (pre-commit formata + auto-commit pusha fix)
- PR: detecta unstaged changes e falha com mensagem clara
- Auto-commit via GITHUB_TOKEN não aciona novos runs (previne loop infinito)
